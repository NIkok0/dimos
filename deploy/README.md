# Deploy artifacts for the dax-agent real-robot thin entrypoint.

This directory lives on the `deploy/dax-agent` branch and carries the
robot-specific packaging that `main` does not need.

## Files

| File | Purpose |
|------|---------|
| `dax-agent.env.example` | GlobalConfig + os.environ template; copy to `.env` on the robot (also on `main`) |
| `dax-agent.service` | systemd unit: auto-start, restart-on-failure, logs to `/var/log/dax-agent/` |
| `install.sh` | one-shot robot install: `uv sync --extra dax-agent` + `.env` + systemd unit |
| `uv.lock.dax-agent` | pinned dependency lock for reproducible robot installs (generated by `uv lock --extra dax-agent`) |

## Robot setup

```bash
git clone -b deploy/dax-agent <dimos-repo> /opt/dax-agent
cd /opt/dax-agent
bash deploy/install.sh
vim .env                      # fill DEEPSEEK_API_KEY / DAX_JOINT_SERVER_URL / ROSBRIDGE_GRPC_TARGET
sudo systemctl start dax-agent
```

## Upgrade

```bash
cd /opt/dax-agent
git fetch && git checkout dax-agent-v1.1   # or `git pull` for latest
uv sync --extra dax-agent                  # re-pin per lockfile
sudo systemctl restart dax-agent
```

## Generating the lockfile

```bash
uv lock --extra dax-agent
cp uv.lock deploy/uv.lock.dax-agent
git add deploy/uv.lock.dax-agent && git commit -m "chore(deploy): relock dax-agent deps"
```
